A note on np.arange and np.linspace¶

Note that the tutorial uses np.arange. While this is fine in some cases, we generally prefer to use np.linspace. np.linspace(a, b, N) divides the interval [a, b] into N points, while np.arange(a, b, s) will step from a to b with a fixed step size s.

One thing to keep in mind is that np.linspace always includes both end points while np.arange will not include the second end point b. For this reason, when we are plotting ranges of values we tend to prefer np.linspace.

The following two functions return the same result, but notice how their parameters differ.

from mpl_toolkits.mplot3d import axes3d

u = np.linspace(1.5*np.pi, -1.5*np.pi, 100)
# This returns an array of evenly spaced numbers from 1.5*pi to -1.5*pi containing 100 elements.
[x,y] = np.meshgrid(u, u)
# This returns an array of coordinate matrices from coordinate vectors. This creates tuples of real numbers.
squared = np.sqrt(x.flatten()**2 + y.flatten()**2)
z = np.cos(squared)
# This line returns an array containing the cosine of all elements from squared. Squared is an array, where each element
# is the square root of the sum of two arrays, x and y, where x and y have been flattened into a 1D array, 
# and then each element inside x and y has been squared. 
z = z.reshape(x.shape)
# This line reshapes the array z into the shape of array x, which is a tuple of ints.
fig = plt.figure()
ax = fig.add_subplot(111, projection='3d')
# These two lines create a 3D subplot with a 1x1 grid, and a note that this is the first subplot.
ax.plot_wireframe(x, y, z, rstride=10, cstride=10)
# This line uses x, y, and z to plot a 3D wireframe, which is a visual presentation for a 3D object. The array row stride and array column stride step sizes are both set to 10.
ax.view_init(elev=50., azim=30)
# This line sets the elevation of the axis to 50 and the azimuth of the axis to 30. The azimuth is the angle between the projected vector and a reference vector on the reference plane.
plt.savefig("figure1.png")
# This line saves the current figure with the filename figure1.png.
